# ReelSpeak — 产品愿景
**版本**: v1.0 | **最后更新**: 2026-07-16

---

## 一、产品定义

ReelSpeak 是一个 **AI 驱动的语言学习平台**。

核心体验：粘贴任意短视频链接 → 自动识别断句 → 逐句复读跟读 → AI 实时发音评分。

不是工具箱。不是 AI 功能集合。是一个完整的**语言习得体验**。

---

## 二、目标用户

**核心用户**：通过短视频学习外语的中文用户，当前重点语种为泰语。

**扩展方向**：日语（日剧/动漫）、韩语（韩剧）、英语，以及将来的西班牙语、法语。

**用户场景**：
- 在手机上刷到一个有趣的泰语视频，想搞懂每句话的意思
- 想通过反复跟读练出地道发音
- 需要 AI 告诉自己哪个音发错了
- 想把学习材料带走（导出字幕、TTS 音频课程）

---

## 三、核心价值主张

| 竞品 | 局限 |
|------|------|
| Pimsleur / ThaiPod101 | 内容固定，无法用自己喜欢的视频学习 |
| 字幕组 APP | 只有字幕，没有发音训练，没有 AI 评分 |
| 语言学习 APP（Duolingo 等） | 内容碎片化，无法沉浸式学习真实语料 |

**ReelSpeak 的差异化**：用自己选的视频，AI 自动生成学习材料，一站式完成听、读、说的完整训练闭环。

## 四、商业化核心原则（不可违反）

这四条原则约束所有与商业化相关的产品决策和技术设计：

### 原则一：用户购买的是学习体验，不是 AI 供应商

用户付费买的是"能帮我学好泰语的能力"，不是"能用 GPT-4"或"能调用 Azure"。

UI 默认不暴露 OpenAI / Gemini / Azure / DeepSeek 等 Provider 名称。

### 原则二：收费对象永远是 Capability（能力），不是 Provider

用户看到的是："转录一段视频 = X Credits"，不是"调用 Groq Whisper = $0.0001/分钟"。

Provider 成本是运营内部数据。Credits 定价由 Pricing Policy 控制，与 Provider 单价波动解耦。

### 原则三：Provider 可以随时替换，不影响用户账户

Groq 涨价 → 切 Azure → 用户的 Credits 余额、历史记录、订阅权益不受任何影响。

任何 Provider 变更对用户完全透明（他们不知道也不需要知道）。

### 原则四：权限由能力定义，不由身份定义

不能有 `if VIP` 或 `if plan == "pro"` 这样的判断散落在代码里。

权限是命名的能力集合（`CanTranscribe`、`CanTTS`、`CanPronunciationAssess`），由 Permission Engine 统一管理。

---

## 五、用户体验原则（同等不可违反）

### 原则五：用户只需要结果，不需要实现细节

所有面向用户的交互，优先呈现“能否完成、完成了什么、还要等多久”。

不要在前端、弹窗、进度提示、错误提示里暴露不必要的技术细节，例如：

- 供应商名称、模型名称、接口格式
- 413 / 429 / verbose_json / word timestamps 之类的实现术语
- 让普通用户无法直接行动的内部故障描述

如果需要等待更久，提示必须尽量短、具体、可预期，例如：

- “正在处理，预计还要 30 秒”
- “视频较长，正在分段处理”
- “这段视频暂时无法处理，请换一个更短的视频”

判断一个方案是否可接受时，优先级顺序是：

1. 用户是否能顺利完成目标
2. 用户是否能理解当前状态
3. 用户是否需要额外学习技术背景

凡是会明显拉长等待时间、增加理解成本、但对用户结果没有直接收益的方案，默认应避免。

## 六、商业模式

**Credits 订阅制**（当前设计阶段）：

| 套餐 | 月 Credits | 定位 |
|------|-----------|------|
| Free | 100 | 体验核心功能（转录 + 翻译），无需注册 |
| Plus | 1,000 | 解锁全部学习能力（TTS、发音评分、导出）|
| Pro | 5,000 | 全能力 + 优先 Provider + 长视频支持 |
| Enterprise | 定制 | 机构、老师定制套餐 |

**Credits 设计逻辑**：
- 用户购买的是 Credits（积分），不是具体 API 调用次数
- Credits 与 Provider 成本隔离：Provider 涨价不影响用户已买的 Credits
- 消费顺序：Subscription Credits → Gift Credits → Paid Credits
- Subscription Credits 月末清零；Gift Credits 有过期时间；Paid Credits 永不过期

**本阶段不实现**：支付（Stripe/微信支付/Apple Pay）留到第三阶段商业化后期。
