# 泰语学习播放器 — 产品路线图

> 最后更新：2026-07-19

## 产品定位

一款通过短视频学泰语的 Web 应用。核心价值：**粘贴视频链接 → 自动识别断句 → 逐句重复跟读 → AI 发音评分**。目标用户：对泰语感兴趣的中文用户。

---

## 当前状态（已完成）

- [x] 本地视频播放 + 逐句重复
- [x] Groq Whisper 语音识别 + Azure Speech 智能校准
- [x] DeepSeek 翻译（泰语→中文）
- [x] 三遍字幕模式（盲听→原文→双语）
- [x] 跟读录音 + Azure 发音评估打分
- [x] 从 TikTok/YouTube 链接下载视频
- [x] 导出带字幕视频 + SRT 字幕文件
- [x] 句子列表、播放速度调节、全屏等基础功能

## 第一阶段：部署上线 + Landing Page

**目标：让自己和朋友能在手机上使用，收集反馈**

### 1.1 部署方案

**选择免费/低成本的部署方式，不花钱或少花钱：**

| 方案 | 费用 | 适合场景 | 限制 |
|------|------|---------|------|
| **Railway.app** | 免费额度 $5/月 | 最推荐，一键部署 Flask | 超出额度需付费 |
| **Render.com** | 免费层 | 简单部署 | 免费层 15 分钟无请求会休眠 |
| **Fly.io** | 免费层 | 全球节点 | 免费配额有限 |
| **Oracle Cloud** | 永久免费 | ARM 实例 | 配置门槛稍高 |
| **自有 VPS** | ¥30-50/月 | 完全控制 | 需要自己维护 |

**推荐起步方案：Railway 或 Render**
- 零运维，git push 自动部署
- 免费额度足够朋友间试用
- 后续用户量增长再迁移到 VPS

### 1.2 部署前需要改的代码

- [ ] **视频存储**：本地文件系统 → 云存储（先用 Cloudflare R2，免费 10GB）
  - 上传视频存到 R2，播放时通过 R2 URL
  - 字幕 JSON 可以先存本地或用 SQLite
- [ ] **环境变量**：所有 API Key 通过环境变量配置，不硬编码
- [ ] **CORS 和安全**：限制 API 访问来源
- [ ] **Gunicorn**：生产环境不用 Flask 自带服务器
- [ ] **静态文件**：前端文件通过 Nginx 或 CDN 分发

### 1.3 Landing Page

**一个漂亮的首页，展示产品核心卖点：**

- 顶部 Hero：大标题 + 一句话描述 + "立即体验"按钮
- 功能展示：3-4 个核心功能卡片（配图标）
  - 粘贴链接，自动识别
  - 逐句重复，三遍学习法
  - AI 发音评分
  - 双语字幕导出
- 使用演示：一个 GIF 或短视频展示操作流程
- 底部 CTA：再次引导用户进入应用

**技术方案：** 用当前的原生 HTML/CSS 写，保持轻量，不引入框架

### 1.4 第一阶段交付物

- [ ] 手机浏览器可正常使用所有功能

## 待打磨功能创意（有价值但暂缓）

### 双语音频课（Bilingual Audio Lesson）

**灵感来源**：用户发现将泰语+中文同行粘入 TTS 后，生成的音频既有泰语又有中文，
盲听效果类似 Pimsleur / ThaiPod101 的双语播客格式，无需看字幕即可理解语义。

**核心体验**：
```
音频：[泰语句] → [中文句] → [泰语句] → [中文句] ...
字幕：泰语播放时显示泰语，中文播放时字幕延续/保持（不切换）
```

**与现有竞品对比**：Pimsleur 年费几百美金，仅音频。本产品可一键从任意文本生成，
且与视频学习流打通，差异化明显。

**已做的预留工作**：
- `SubtitleFile.type` 字段已加入，默认 `"standard"`，未来新类型写 `"bilingual_audio"`
- 老课程 JSON 不含 `type` 字段，读取时默认 `"standard"`，向后兼容

**实现时需要解决的问题**：
1. TTS 生成：每行产出两段音频（泰语用泰语声音，中文用中文声音）
2. Segment 时长：延伸覆盖泰语+中文音频之和
3. 播放器：读到 `type=bilingual_audio` 时，隐藏影子跟读/发音评分按钮
4. 课程列表：双语音频课显示特殊标记，让用户有预期
5. 卡拉OK高亮：中文音频播放期间整句保持高亮，不做词级动画

**触发条件（何时重启这个功能）**：
- 用户量到一定规模，有明确的"纯听学习"需求反馈
- TTS 课程功能已作为独立模块有稳定用户群

## 风险与注意事项

1. **版权风险**：下载和分发 TikTok/YouTube 视频可能涉及版权问题
   - 缓解：用户自行提供链接，平台不存储和分发原始视频
   - 长期：与内容创作者合作，获取授权

2. **API 成本失控**：Azure 发音评估是主要成本
   - 缓解：免费用户限制使用次数，付费用户也设日上限
   - 备选：后续调研更便宜的发音评估方案

3. **技术债务**：当前无框架的前端在功能增多后难以维护
   - 缓解：第二阶段结束前评估是否需要引入前端框架
   - 原则：功能 > 架构，先跑通再优化

4. **用户获取**：产品好但没人知道
   - 缓解：利用泰语学习社群推广，提供免费试用
   - 内容营销：在小红书/抖音分享泰语学习技巧，引流到产品
